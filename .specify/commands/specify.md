Crée une spécification complète pour la fonctionnalité demandée.

Étapes :
1. Analyser le besoin décrit par l'utilisateur
2. Consulter `constitution.md` pour les principes du projet
3. Rédiger la spec dans `specs/<feature-name>/spec.md` selon le template :
   - Contexte et objectif
   - Critères d'acceptance
   - User stories
   - Exigences fonctionnelles
   - Contraintes techniques
   - Dépendances
4. Valider la spec avec la checklist qualité
