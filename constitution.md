# Constitution — blueprint-test

This document defines the fundamental principles of the project. It is read before any specification or implementation.

This project is a clone of **Blueprint AI Stack** (master template).
All validation rules from `blueprint-core.md` apply.

## Development Principles

- **Spec-Driven:** every implementation is preceded by a specification.
- **Traceability:** every artifact (spec, ticket, commit) carries a unique identifier and explicit links.
- **Human Review:** no spec or code is validated without peer review.

## Code Quality

- Tests are mandatory for every feature.
- Lint before every commit.
- A PR must be approved before merge.

## Project Management

- Plane is the source of truth for progress.
- n8n syncs statuses between Plane and local tools.
- Graphify maps project dependencies.

## Blueprint Structure

- `blueprint-core.md` — core rules and orchestration
- `phase-1.md` — phase 1 definition
- `tool-status.md` — tool operational status
- `constitution.md` — these principles (this file)
